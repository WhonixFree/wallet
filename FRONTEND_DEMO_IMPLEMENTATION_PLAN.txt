Implementation Goal

Build a polished frontend-only demo version of the local Windows desktop crypto portfolio tracker.

The goal of this phase is not backend completeness.
The goal is to make the application look fully functional, visually credible, and smooth enough for marketing videos, product walkthroughs, and teaser recordings.

The build should:

look production-ready on camera
feel responsive and alive
show realistic portfolio data
support believable user flows
remain compatible with the future real architecture
Scope of This Plan

This plan covers only the demo-first frontend implementation.

Included in this phase:

Electron desktop shell
React renderer
routing and navigation
app layout
seeded mock data
async mock services
realistic loading / success / error states
polished UI for main product flows
demo-only assistant chat UI
demo-only import preview UI
local frontend state management
visual behavior suitable for recorded walkthroughs

Excluded from this phase:

real Go backend
real SQLite persistence
real preload-to-backend production data flow
real CoinGecko integration
real chain explorer integration
real file parsing
real markdown file IO
real notifications
real AI integration
Architectural Intention

This phase must preserve the long-term architecture direction.

Important constraints:

Electron remains the desktop shell
React remains the UI layer
mock services should imitate future backend-facing service boundaries
data shapes should stay close to future domain and contract structures
renderer should not become the permanent source of truth for business logic

The demo build is a temporary product shell, not a replacement for the real backend-driven implementation.

Success Criteria

This phase is successful when:

the app can be opened and navigated like a complete product
every major screen looks intentionally designed
the app supports a clean demo flow from dashboard to assistant chat
no major surface looks empty, broken, or obviously unfinished
UI interactions feel believable even when fully mocked
Codex can continue feature work without restructuring the app later
Implementation Strategy

Use a vertical-slice frontend strategy.

Do not start by building every page at once in shallow form.
Instead, build a complete demo shell and then make each major feature visually complete one by one.

Recommended order:

repo/bootstrap and layout
mock data foundation
service layer
dashboard
wallets flow
groups flow
alerts flow
calendar flow
search
notes
transfers

polish pass
demo recording pass
